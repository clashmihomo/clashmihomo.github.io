ONVIF开发（2）vs2013编译openssl库

作者最近在windows下基于gsoap开发一个onvif客户端，winsows平台下gsoap鉴权一般采用自带的鉴权函数，但必须调用openssl库，下面将详细介绍如何在windows平台下利用vs2013编译openssl库：
1、下载openssl源码，作者下载的版本是openssl-1.0.1e，1.0及以上版本可能会失败，原因是部分编译指令不支持。

2、解压openssl-1.0.1e.tar.gz到S盘（作者解压到的目录）
3、下载Perl，根据系统提示安装即可，安装成功后添加环境变量，在系统变量Path一栏添加 C:\Perl64\bin;（C:\Perl64\ 为作者的安装目录），重启系统
4、新建目录 S:\openssl_lib\x86 存放编译后的库文件
5、打开2013控制台：开始-所有程序—Microsoft visual studio2013——Visual Studio Tools——
6、VS2013 x86 本机工具命令提示
7、执行cd /d S:/openssl-1.0.1e（进入openssl-1.0.1e解压目录，此处因为是从C盘跨越到S盘，所以加上/d）

8、执行perl Configure VC-WIN32 no-asm --prefix=S:\openssl_lib\x86 （存放编译后的库文件目录）

9、执行ms\do_ms

10、执行指令nmake -f ms\ntdll.mak（编译动态库，大约几分钟，耐心等待…）
11、执行指令nmake -f ms\nt.mak （编译静态库）

装载openssl到指定的目录S:\opensll_lib\x86中
12、执行指令nmake -f ms\ntdll.mak install
13、执行指令nmake -f ms\nt.mak install

编译结束，库目录包含bin、include、lib、ssl 四个文件夹
