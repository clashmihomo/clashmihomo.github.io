一、Quantumult X 简介：不只是代理工具

Quantumult X 最初被广泛认知为一款 HTTP/HTTPS 代理工具，支持多种代理协议和复杂的规则配置。然而，它的功能远不止于此。通过内置的脚本引擎，用户可以使用 JavaScript 编写自定义脚本，实现网络请求的修改、数据捕获、定时任务等多种高级功能。这种高度可扩展的设计，使得 Quantumult X 从单纯的代理工具演变为一个综合性的网络管理平台。

对于许多用户来说，定时脚本可能是最实用的功能之一。无论是定时检查服务状态、自动签到，还是定期抓取数据并推送到其他平台，定时脚本都能在无需人工干预的情况下完成这些任务，极大地提升了效率和便利性。

二、定时脚本概述：何为定时脚本，为何需要它？

定时脚本，顾名思义，是指在特定时间或时间间隔自动执行的脚本。在 Quantumult X 中，用户可以通过配置定时任务，让脚本在后台按计划运行。这种自动化能力为用户打开了无限可能：

  1. 定期执行网络请求：例如，每天自动访问某个 API 接口，获取最新数据或执行签到操作。
  2. 自动化管理任务：比如定时清理缓存、更新配置或重启服务。
  3. 监控与告警：通过定时检查网络连接或服务状态，及时发现问题并通知用户。

这些应用场景不仅节省了用户的时间，还减少了因人为疏忽导致的错误。更重要的是，定时脚本允许用户根据自身需求定制功能，真正实现个性化网络管理。

三、如何编写与配置定时脚本

要使用 Quantumult X 的定时脚本功能，用户需要具备基本的 JavaScript 知识。以下是详细的步骤指南：

步骤 1：打开脚本编辑器

首先，在 Quantumult X 应用中进入“设置”菜单，找到“脚本”选项。点击进入后，用户可以看到脚本编辑界面，这里可以编写、修改和管理所有自定义脚本。

步骤 2：编写基本脚本

在脚本编辑器中，用户可以开始编写 JavaScript 代码。例如，一个简单的脚本可以输出日志信息：

javascript console.log('Hello, Quantumult X!');

这段代码虽然简单，但可以用来验证脚本是否正常运行。在实际应用中，用户可以编写更复杂的逻辑，比如发送 HTTP 请求、解析响应数据、判断条件并执行相应操作。

步骤 3：保存脚本

编写完成后，务必检查代码是否存在语法错误。确认无误后，保存脚本。Quantumult X 会自动验证脚本的合法性，如果存在错误，会提示用户修改。

步骤 4：配置定时任务

保存脚本后，需要配置定时任务来触发脚本执行。回到设置菜单，找到“任务管理”选项，选择“添加定时任务”。在这里，用户可以设置任务的触发时间、执行频率（如每天、每周或特定时间点），并关联之前保存的脚本。

例如，用户可以设置一个每天上午 9 点执行的脚本，用于检查某个服务的状态并发送通知。通过灵活配置定时任务，用户可以满足各种场景的需求。

四、定时脚本的应用场景

定时脚本的功能不仅限于简单的自动化任务，还可以应用于更复杂的场景中。以下是几个典型例子：

  1. 网络请求与数据抓取：用户可以通过定时脚本定期访问特定网站或 API，抓取所需数据并保存到本地或推送到其他平台。例如，定时获取天气预报、股票价格或新闻头条。

  2. 服务监控与告警：对于需要高可用性的服务，用户可以编写脚本定时检查服务状态。如果发现异常，脚本可以自动发送邮件或消息通知，帮助用户及时处理问题。

  3. 自动化管理操作：例如，定时清理缓存文件、更新代理规则或重启网络连接。这些操作可以保持系统运行在最佳状态，避免因资源占用过多而导致的性能下降。

  4. 个性化签到与任务执行：许多网站和应用提供了每日签到功能，通过定时脚本，用户可以自动完成这些任务，无需手动操作。这不仅节省时间，还能确保不会错过任何奖励。

在使用 Quantumult X 定时脚本的过程中，用户可能会遇到一些常见问题。以下是这些问题的解答：

  1. 定时脚本可以设置多长的运行时间？
    Quantumult X 允许用户灵活配置定时任务的执行频率，但没有设置最大持续时间。用户可以根据需求选择合适的时间间隔，但需注意过于频繁的任务可能会对设备性能产生一定影响。

  3. 脚本中的错误会影响其他脚本吗？
    是的，如果一个脚本存在错误，可能会导致后续任务无法正常执行。因此，建议用户在部署脚本前进行充分测试，确保代码的稳定性和可靠性。

    目前，Quantumult X 仅支持 JavaScript 编写的脚本。对于熟悉其他语言的用户，可能需要一定的学习成本，但 JavaScript 的灵活性和强大功能足以满足大多数需求。

  5. 能否共享脚本给其他人使用？
    是的，用户可以将脚本代码复制并分享给其他 Quantumult X 用户。只需将代码粘贴到脚本编辑器中，配置相应的定时任务即可使用。这种共享机制促进了用户之间的交流与合作，丰富了脚本生态。

FAQ

SSR 如何选择合适混淆提高稳定性？
选择混淆方式时应根据网络环境和封锁程度，例如 tls1.2_ticket_auth 可模拟 HTTPS 流量。合理组合协议和混淆可提升节点稳定性和抗封锁能力，确保长期科学上网体验。
Mihomo 的 Web Dashboard 有哪些功能？
Mihomo 的 Web 控制面板允许用户可视化管理节点、分流规则、日志查看与系统状态监控。可通过浏览器访问 Dashboard 地址进行远程管理，方便多设备统一控制。
Matsuri 的节点自动测速功能如何配置？
Matsuri 可以设置测速周期和节点列表，程序会自动测试延迟和速度并排序。用户可根据测试结果选择最佳节点，提高网络访问速度和使用体验。
Sing-box 是什么？
Sing-box 是新一代多协议代理框架，支持 VMess、VLess、Trojan、Hysteria、ShadowTLS 等协议。它比传统 V2Ray 更轻量化、模块化，并针对移动端和 Linux 系统进行了性能优化，受到开发者和高级用户的欢迎。
Trojan TLS 如何避免被 DPI 检测？
通过选择真实可访问域名并使用有效证书，Trojan 流量可完全模拟 HTTPS 请求。隐藏真实流量特征，降低被封锁和 DPI 检测风险，保证节点长期可用。
Hysteria2 协议的主要特点是什么？
Hysteria2 是新一代基于 QUIC 的高速代理协议，支持自适应拥塞控制和零 RTT 连接。相比传统 TCP 协议，它能在高延迟或丢包环境下保持稳定速度，特别适合跨境访问或实时传输场景。
Mellow 是否支持自定义 DNS？
支持。Mellow 的配置文件中可设置 DNS 服务器地址及模式（如 DoH、DoT）。用户可使用安全 DNS，如 Cloudflare 1.1.1.1 或 Google DNS，以减少污染风险。
Nekoray 的配置文件在哪里保存？
Nekoray 的配置文件默认保存在程序目录下的 `config.json` 中。用户可手动备份此文件以迁移配置，也可使用导入导出功能在不同设备间同步节点数据。
Mellow 支持哪些平台？
Mellow 是基于 Electron 的跨平台代理工具，支持 Windows、macOS 和 Linux。它通过 TUN 模式接管全局流量，并支持 Clash 规则和订阅格式，非常灵活。
PassWall2 的节点测速结果如何影响分流？
PassWall2 会根据节点延迟和成功率自动调整策略组选择最优节点。测速结果直接影响程序选择出口，从而提高连接速度和稳定性。
